使用Docker实现MySql的主从

2019-07-04

前言

先说一下为什么会在docker中进行mysql的主从安装!

之前一直在虚拟机或者云主机中进行环境配置,但是介于有时候网络的原因或宿主机配置不是很够的情况下去做这个就有点尴尬了.

docker的好处就无需多说了,简单方便,谁用谁知道

废话不多说,先来看一下环境

OS:

2015MBP
Docker Version:18.09.2
MYSql:5.7.17

MySql主从复制的方式:

MySQL 5.6 开始主从复制有两种方式：基于日志（binlog）和 基于 GTID（全局事务标示符）。

本文只涉及基于日志 binlog 的 主从配置。

配置

docker pull mysql:5.7

起master库
docker run --name mdb1 -e MYSQL_ROOT_PASSWORD=secret -p 13307:3306 -v ~/docker/mysql_data:/var/lib/mysql -d IMAGE ID

起slave库
docker run --name sdb1 -e MYSQL_ROOT_PASSWORD=secret -p 13308:3306 -d IMAGE ID

进入容器
docker exec -it mdb1 bash

修改主库配置文件
vim /etc/mysql/my.cnf

//用于标示该数据库为主库 ,从库配置文件需修改为小于该value的值
server_id=1


创建从数据库的用户,用于连接同步数据
CREATE USER 'sync'@'%' IDENTIFIED BY 'sync';

更新权限
GRANT REPLICATION SLAVE ON *.* TO 'sync'@'%';	

查看主库状态,记住File、Position的值，在Slaver中将用到
show master status;

进入从库容器
docker exec -it sdb1 bash

mysql -u root -p

//设置主库的连接
change master to
master_host='主库地址',
master_user='sync',
master_password='sync',
master_port=端口号,
master_log_file='binlog.000005',//file 的值
master_log_pos=1686;//Position的值

开启同步
start slave;


查看状态
show slave status;

后语

其实配置还是很方便的,这个只是其中的简单配置,下次有时间使用DockerFile + Docker Compose 来做管理
