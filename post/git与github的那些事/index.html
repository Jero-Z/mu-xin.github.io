Git与GitHub的那些事

2019-07-04

多快好省,简单粗暴

1.阐述功能

> git :

Gitis a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

⬆️引述于官方首页,简单来说Git是目前世界上最先进的分布式版本控制系统（没有之一）。


> github :

GitHub, Inc. is a US-based global company that provides hosting for software development version control using Git. It is a subsidiary of Microsoft, which acquired the company in 2018 for US$7.5 billion. It offers the distributed version control and source code management (SCM) functionality of Git, plus its own features. It provides access control and several collaboration features such as bug tracking, feature requests, task management, and wikis for every project.

> 最大同人社区（没有之一）

2.上手测试

1. git init//初始化仓库 
2. git add //添加文件到暂存区
3. git commit //生成快照
4. git remote add origin "git repository address"//添加远端仓库地址
5. git push 推送快照到远端仓库

3.背后细节

> 上面的那些操作能满足我这种面向互联网开发程序猿吗?能!也不能
如果简单的在项目中运用的话,做为一个初级开发其实足够了,再学习一些分支技巧知道提交merge request 的确就足够了,但是作为一个胸有大痣的我怎么可能这么简单的就满足呢?那么接下来就慢慢的深入她,揭开她的面纱吧....

  1. git init

    命令只做一件事，就是在项目根目录下创建一个.git子目录，用来保存版本信息。

  2. git add

    这个命令其实是简化了两个操作步骤,

    git hash-objecgt -w fileName

    git update-index --add --cacheinfo

    那么问题来了,这两个兄弟又是做了些什么呢?

    git hash-objecgt -w fileName //生成hash值

    git hash-object -w README.md >1439efeb9f724f332c0d935905b12c26ce8009c1`

    ll .git/objects/14 > 39efeb9f724f332c0d935905b12c26ce8009c1`

    git caat-file -p 1439efeb9f724f332c0d935905b12c26ce8009c1 > # git-flow-process Hello Word`

    git update-index --add --cacheinfo//将hash后的文件添加到暂存区

    git update-index --add --cacheinfo 664(权限) 1439efeb9f724f332c0d935905b12c26ce8009c1 (hash值) README.md(文件名)

    git ls-files --stage

    100644 1439efeb9f724f332c0d935905b12c26ce8009c1 0 README.text

    ok,装逼效果拉满了是吧?其实⬆️面的命令就等同于git add 和 git status

  3. git commit

    提交文件,生成本地快照

    简单的来说就是本地生成一份快照,或者说归档文件,嗯!就这么简单,可使用 -m参数添加生成快照时的备注信息,以便于后期的浏览和回滚

    一个人玩其实很没有意思的,大家一起来才基情满满嘛!怎么办呢?当然是 选择原谅他!

    推送到全球最大的同性交友网站咯,万一那个同志看上你   –)写的东西了呢?

    git remote add origin 你的远端仓库地址

    添加远端仓库地址

    当然了,推送前要标注好自己的身份,好让别人知道你是谁

    git config user.name "用户名"

    git config user.email "Email 地址"

    git push -u origin master > 推送到远端分支,完成提(我敢保证实际项目开发中你肯定没有直接推送master的权限,那怎么办呢?只能引入分支模块了呗,这里不多赘述)

    接下来就是github线上进行操作的时候了!

    1. 打开浏览器
    2. 登陆github
    3. 点击项目仓库
    4. 点击Pull request
    5. create pull request
    6. 选择review|assignees等
    7. 点击提交(完毕!)

你以为就这样就结束了吗?

少年醒醒,开始项目发布了!

git checkout master//发布版本肯定是在主分支上进行了!

git tag version(v0.0.1)

该命令会在当前最新版本进行发布,如果想发布过去时间段的怎么办?

git tag version(v0.0.1) 3e7cc97bd05d32759bd3c6b64c6fbb89b9aaddbb

git tag//查看当前所有的tag

当然了,现在只是在本地进行了打标签,如果想发布到github该如何操作呢?

git push origin v0.0.1 | git push origin --tags

第一个命令推送指定tag到远端仓库,第二个命令推送本地所有tag到远端

就这样简单方便的成功发布了第一版本的项目!
